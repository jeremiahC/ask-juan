How It Works

One entry point. Full team coverage.

Tell Juan what you need — he reads context, picks the right specialist, and injects lean project info so they can hit the ground running.

📖

Context-aware

Every agent reads .juan/context.md — stack, phase, constraints — before doing anything. No re-explaining your project.

🎯

Precise routing

Juan matches intent to specialist: PM ask → Rica. Architecture question → Carlo. Frontend bug → Nico. Never the wrong agent.

🔗

Chained handoffs

Rica's PRD feeds Carlo's plan, which gates Nico and Rex. Tina triggers Lara, Dino, and Bea. Outputs always flow to the right next step.

⚡

Parallel execution

Full-feature work spawns Rica + Mika + Carlo in parallel, then Nico + Rex concurrently. QA runs Lara + Dino + Bea together.

🔒

Security gated

Carlo's security audit runs before QA on any feature touching auth, payments, PII, or external APIs. Not optional.

✅

Quality signed off

Tina synthesizes Dino's test report, Lara's coverage, and Bea's manual review into one go/no-go decision before merge.
